The app crashes when HealthKit authorization is denied on iOS 18.4. The crash log shows an unexpected nil in SleepDataManager.fetchSleepData(). Can you fix this?

ASSISTANT
I'll investigate the crash. Let me first look at the SleepDataManager implementation.
Found the issue. The fetchSleepData() method force-unwraps the HealthKit store without checking authorization status. On iOS 18.4, denied authorization returns nil instead of an empty result.
guard let store = healthStore else { logger.warning("HealthKit store unavailable") return [] } let status = store.authorizationStatus(for: sleepType) guard status == .sharingAuthorized else { return [] }
I've added a guard clause that checks both store availability and authorization status before attempting to fetch sleep data. This prevents the crash on iOS 18.4 when the user denies HealthKit access.
